# MIG4004 — System.DirectoryServices / Active Directory

| | |
| --- | --- |
| **Category** | Runtime failures |
| **Severity** | High |
| **Effort** | Medium |
| **Confidence tier** | 2 — Probable |

## What it detects

A `using System.DirectoryServices;` (or nested) directive.

## Why it matters

`System.DirectoryServices` is Windows-only on modern .NET and throws
`PlatformNotSupportedException` elsewhere.

## How to fix it

For cross-platform LDAP access use `System.DirectoryServices.Protocols`, which is supported
on Linux. For authentication, consider an identity provider (Entra ID / OIDC) instead of
direct AD calls.

## Known false positives

Low. The namespace is specific. Detection matches both `using` directives and fully-qualified
references.

## Target platform

This is a **Windows lock-in**, not a gone-everywhere API. When you scan with a Windows target
(`--target net10.0-windows`), MigrationScan treats this finding as *satisfied*: it stays listed
for completeness but is excluded from the counts, effort estimate, and `--fail-on`. It only
becomes migration cost if the app must also run off Windows.
