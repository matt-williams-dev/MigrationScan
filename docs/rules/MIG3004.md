# MIG3004 — WCF service host (server side)

| | |
| --- | --- |
| **Category** | Blocking frameworks |
| **Severity** | High |
| **Effort** | Large |
| **Confidence tier** | 2 — Probable |

## What it detects

Use of `ServiceHost` in C# source — the server-side entry point for hosting a WCF service.

## Why it matters

Server-side WCF is not part of modern .NET. The client stack (`System.ServiceModel.*`) is
supported, but hosting a service is not.

## How to fix it

- **CoreWCF** — the community successor; closest to a lift-and-shift for existing contracts.
- **gRPC** — if you can change the wire protocol and want a modern, cross-platform RPC stack.
- **Minimal API / Web API** — if the service is effectively HTTP.

## Known false positives

Tier 2. Matches the identifier `ServiceHost` regardless of namespace, so an unrelated type
named `ServiceHost` would also match. Verify the reference is `System.ServiceModel.ServiceHost`.
