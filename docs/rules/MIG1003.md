# MIG1003 — Target framework below 4.6.2

| | |
| --- | --- |
| **Category** | Project and build |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 1 — Certain |

## What it detects

A project whose target framework is a .NET Framework version **below 4.6.2** — read from
`<TargetFrameworkVersion>` (e.g. `v4.5`) or a `net4x` moniker (e.g. `net461`). Modern targets
(`net5.0`+, `netcoreapp*`, `netstandard*`) are not flagged.

## Why it matters

.NET Framework 4.6.2 is the practical floor for migration: earlier versions lack full
.NET Standard 2.0 support, which the migration path and much of the modern package ecosystem
rely on.

## How to fix it

Retarget the project to at least 4.6.2 (ideally 4.8) and rebuild. This is usually a low-risk
retarget-and-rebuild step, and it should happen before the move to modern .NET.

## Known false positives

None. The version is read directly from the project file.
