# MIG2001 — Package has no version supporting the target framework

| | |
| --- | --- |
| **Category** | Dependencies |
| **Severity** | High |
| **Effort** | Medium |
| **Confidence tier** | 1 — Certain |

## What it detects

A referenced package (from `packages.config` or `PackageReference`) that appears in the
bundled catalog of packages known to have no version supporting modern .NET — for
example `Microsoft.AspNet.Mvc`, `Microsoft.AspNet.WebApi`, or Microsoft Enterprise Library.

The offline catalog lives at
[`src/MigrationScan.Core/Data/incompatible-packages.json`](../../src/MigrationScan.Core/Data/incompatible-packages.json).
The `--online` path (Phase 6) will supplement it with live nuget.org compatibility lookups.

## Why it matters

If a dependency has no modern-.NET-compatible version, it must be replaced or removed
before the project can move off .NET Framework. These are often the highest-impact items
in an assessment.

## How to fix it

Replace the package with a modern successor (the finding message names a suggested
replacement) or remove the dependency. Some have direct equivalents (e.g. ASP.NET MVC 5 →
ASP.NET Core MVC); others require re-architecture.

## Known false positives

Low. The catalog is a curated list of packages that are genuinely Framework-only. Matching
is by exact package ID (case-insensitive). A package not in the catalog is not flagged in
the offline path even if it is in fact incompatible — that gap is closed by `--online`.
