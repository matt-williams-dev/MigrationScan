# MIG5001 — ConfigurationManager.AppSettings usage

| | |
| --- | --- |
| **Category** | Configuration |
| **Severity** | Low |
| **Effort** | Small |
| **Confidence tier** | 2 — Probable |

## What it detects

A member-access expression `ConfigurationManager.AppSettings` in C# source, matched on the
Roslyn syntax tree without a resolved compilation.

```csharp
var env = ConfigurationManager.AppSettings["Environment"];
```

## Why it matters

On .NET Framework, `System.Configuration.ConfigurationManager` is built in. On modern .NET
it is not: reading `AppSettings` requires adding the `System.Configuration.ConfigurationManager`
NuGet package, or migrating configuration to `Microsoft.Extensions.Configuration`.

## How to fix it

- **Minimal:** add the `System.Configuration.ConfigurationManager` package and keep using
  `app.config`. Works, but keeps the old configuration model.
- **Preferred:** migrate to `Microsoft.Extensions.Configuration` — `appsettings.json`,
  environment variables, and the options pattern — which is the modern .NET idiom.

## Known false positives

This is a **Tier 2 (probable)** finding. It matches on syntax alone, so a member named
`AppSettings` on some unrelated `ConfigurationManager`-named type would also match. The
finding is reported as *probable* for exactly this reason — verify before acting on it in
a codebase that defines its own `ConfigurationManager`.
