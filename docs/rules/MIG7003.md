# MIG7003 — System.Data.OleDb on non-Windows

| | |
| --- | --- |
| **Category** | Data access |
| **Severity** | Medium |
| **Effort** | Medium |
| **Confidence tier** | 2 — Probable |

## What it detects

Use of the `System.Data.OleDb` namespace.

## Why it matters

The OLE DB provider is Windows-only on modern .NET (via the `System.Data.OleDb` package) and is
not available on other platforms.

## How to fix it

Use a native, cross-platform data provider where possible — `Microsoft.Data.SqlClient` for SQL
Server, `Npgsql` for PostgreSQL, and so on.

## Known false positives

Low. The namespace is specific. Detection matches imports and fully-qualified references.
