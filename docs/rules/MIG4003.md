# MIG4003 — System.Management / WMI

| | |
| --- | --- |
| **Category** | Runtime failures |
| **Severity** | High |
| **Effort** | Medium |
| **Confidence tier** | 2 — Probable |

## What it detects

Use of the `System.Management` namespace (WMI).

## Why it matters

On modern .NET, `System.Management` is Windows-only (via the `System.Management` package) and
throws `PlatformNotSupportedException` on other platforms — compiles fine, fails at runtime.

## How to fix it

If the app must run cross-platform, replace WMI queries with platform-appropriate APIs. On
Windows-only deployments the dependency is still fine (add the package).

## Known false positives

Low. The namespace is specific. Detection matches imports and fully-qualified references.
