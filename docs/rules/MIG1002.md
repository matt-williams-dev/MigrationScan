# MIG1002 — packages.config instead of PackageReference

| | |
| --- | --- |
| **Category** | Project and build |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 1 — Certain |

## What it detects

A `packages.config` file next to the project. This is the legacy NuGet dependency
format; SDK-style projects require `PackageReference` instead.

## Why it matters

`packages.config` is incompatible with SDK-style projects and does not support
transitive dependency restore. It must be migrated as part of moving to modern .NET.

## How to fix it

Migrate to `PackageReference`. Visual Studio offers an in-place migration
(right-click the project → *Migrate packages.config to PackageReference*). After
migrating, delete `packages.config` and remove the per-package `<Reference>` items
that pointed into the `packages` folder.

## Known false positives

None. The file is either present or it is not.
