# MIG2002 — Package marked deprecated on nuget.org

| | |
| --- | --- |
| **Category** | Dependencies |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 1 — Certain |
| **Requires** | `--online` |

## What it detects

A referenced package (from `packages.config` or `PackageReference`) that its maintainers have
marked **deprecated** on nuget.org. This is an **online-only** rule: it reports nothing unless
you pass `--online`, because the default path makes no network calls.

## Why it matters

A deprecated package stops receiving fixes — including security fixes — and is a migration
risk even when it still runs. Deprecations often point at a supported successor, so this is
usually a good change to fold into the migration rather than defer.

## How to fix it

Move to the replacement the maintainers name (surfaced in the finding message when nuget.org
provides one) or to an actively-maintained alternative. If the package is deprecated but you
must keep it, record that as an accepted risk.

## Known false positives

Low. Deprecation status is an authoritative fact reported by nuget.org, hence Tier 1. The one
caveat is timeliness: because it reflects live nuget.org state, the same scan can change over
time as packages are deprecated (or un-deprecated) — which is why it is gated behind `--online`
and never part of the deterministic default path.
