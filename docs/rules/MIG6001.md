# MIG6001 — BinaryFormatter

| | |
| --- | --- |
| **Category** | Serialization and security |
| **Severity** | Blocker |
| **Effort** | Large |
| **Confidence tier** | 2 — Probable |

## What it detects

Use of the identifier `BinaryFormatter`
(`System.Runtime.Serialization.Formatters.Binary.BinaryFormatter`).

## Why it matters

`BinaryFormatter` is **removed in .NET 9** and throws when invoked. It was also a long-standing
remote-code-execution risk. Any code path that serializes or deserializes with it is a hard
blocker.

## How to fix it

Replace it with a safe serializer: `System.Text.Json`, MessagePack, or protobuf. Because the
on-disk/on-wire format changes, persisted `BinaryFormatter` data needs a migration path.

## Known false positives

Low. `BinaryFormatter` is a specific, unambiguous type name. Reported as *probable* only
because it is matched on syntax (e.g. a `using` alias could rename it).
