# MIG4013 — P/Invoke to a Windows system DLL

| | |
| --- | --- |
| **Category** | Runtime failures |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 2 — Probable |

## What it detects

A `[DllImport(...)]` attribute whose target is a **known Windows system library** —
`kernel32`, `user32`, `advapi32`, `gdi32`, `shell32`, `ole32`, `ws2_32`, and similar.

```csharp
[DllImport("kernel32.dll", SetLastError = true)]
static extern IntPtr CreateEvent(IntPtr attrs, bool manualReset, bool initialState, string? name);
```

P/Invokes to bespoke native libraries (e.g. a vendor's `scanner_sdk.dll`) are **not** flagged
by this rule — those are the caller's own native surface, not a Windows-platform dependency.

## Why it matters

This is a **Windows lock-in**, not a gone-everywhere API. The P/Invoke works on modern .NET
when the project targets Windows (`net10.0-windows`); it simply has no equivalent off Windows,
where the call will fail at runtime.

There is a valid migration path: a Windows-only app can keep these calls by targeting
`net-windows`. They only become work when the app must run cross-platform.

## How to fix it

- **Staying on Windows:** target `net-windows`; the call keeps working, no change needed.
- **Going cross-platform:** guard the call with `OperatingSystem.IsWindows()` and provide a
  non-Windows path, or replace it with a cross-platform .NET API (many old Win32 P/Invokes now
  have managed equivalents — `EventWaitHandle` for `CreateEvent`, etc.).

## Known false positives

Tier 2: the rule matches on the DLL name in the `[DllImport]` string, which is reliable, but a
custom library that happens to be named like a system DLL would match. Confirm the target is the
real Windows system library before acting.

## Target platform

When you scan with a Windows target (`--target net10.0-windows`), MigrationScan treats this
finding as *satisfied*: it stays listed for completeness but is excluded from the counts,
effort estimate, and `--fail-on`. It only becomes migration cost if the app must also run off
Windows.
