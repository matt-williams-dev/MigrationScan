# MIG8002 — Encoding.Default behavior change

| | |
| --- | --- |
| **Category** | Globalization and encoding |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 2 — Probable |

## What it detects

A `Encoding.Default` member access.

## Why it matters

On .NET Framework, `Encoding.Default` is the system's ANSI code page. On modern .NET it is
**always UTF-8**. Code that read or wrote bytes assuming ANSI will silently produce different
results after migration — no exception, just wrong bytes.

## How to fix it

Make the intended encoding explicit wherever bytes cross a boundary: use `Encoding.UTF8`, or
a specific code page via `Encoding.GetEncoding(...)` (see [MIG8003](MIG8003.md)).

## Known false positives

Low. `Encoding.Default` is specific; the rule matches the `Encoding` receiver and `Default`
member. Whether the change actually matters depends on whether the data was really ANSI.
