# MIG7001 — System.Data.SqlClient

| | |
| --- | --- |
| **Category** | Data access |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 2 — Probable |

## What it detects

A `using System.Data.SqlClient;` directive.

## Why it matters

`System.Data.SqlClient` is in maintenance mode and no longer receives new features or full
support. The supported successor is `Microsoft.Data.SqlClient`.

## How to fix it

Switch the package and namespace to `Microsoft.Data.SqlClient`. **Watch the connection
string:** version 4.0+ defaults to `Encrypt=true`, which can break connections to servers
without a trusted certificate (see [MIG7002](https://github.com/matt-williams-dev/MigrationScan)).

## Known false positives

Low. `System.Data.SqlClient` is a specific namespace. Detection matches both `using`
directives and fully-qualified references (e.g. `new System.Data.SqlClient.SqlConnection()`).
