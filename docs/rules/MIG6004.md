# MIG6004 — Code Access Security attributes

| | |
| --- | --- |
| **Category** | Serialization and security |
| **Severity** | Medium |
| **Effort** | Medium |
| **Confidence tier** | 2 — Probable |

## What it detects

Use of a Code Access Security attribute: `SecurityPermission`, `PermissionSet`, or
`PrincipalPermission` (with or without the `Attribute` suffix).

## Why it matters

Code Access Security is not honored on modern .NET. These attributes are obsolete; at best
they are ignored, and the security guarantee they implied no longer holds.

## How to fix it

Remove the CAS attributes and enforce authorization at the application layer (e.g. ASP.NET
Core authorization policies, role checks) where the guarantee actually matters.

## Known false positives

Tier 2. Matches the attribute's simple name, so an unrelated attribute named, say,
`PrincipalPermission` from another library would match. Confirm it is the
`System.Security.Permissions` attribute.
