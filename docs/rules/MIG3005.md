# MIG3005 — .NET Remoting

| | |
| --- | --- |
| **Category** | Blocking frameworks |
| **Severity** | Blocker |
| **Effort** | Blocker |
| **Confidence tier** | 2 — Probable |

## What it detects

A `using` directive for a `System.Runtime.Remoting` namespace.

## Why it matters

.NET Remoting was removed entirely from modern .NET and has no drop-in replacement. Code
that depends on it cannot run until the remoting boundary is redesigned.

## How to fix it

Replace the remoting boundary with a supported transport: gRPC, WCF on CoreWCF, or an HTTP
API. This is an architectural change, not a mechanical port.

## Known false positives

Low. The `System.Runtime.Remoting` namespace is specific to Remoting. Detection is by
`using` directive, so fully-qualified usage without a `using` would be missed (rare).
