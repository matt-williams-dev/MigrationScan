## What it detects

Use of the identifiers `Registry` or `RegistryKey` (`Microsoft.Win32.Registry`).

## Why it matters

The Windows Registry does not exist on other platforms. `Microsoft.Win32.Registry` throws
`PlatformNotSupportedException` off Windows ‚Äî compiles fine, fails at runtime on Linux.

## How to fix it

Move the configuration or state the registry held to a cross-platform store: a config file,
environment variables, or a database. On Windows-only deployments the registry is still fine.

## Known false positives

Tier 2, and this is the canonical example (spec ¬ß5): a class of your own named `Registry` or
`RegistryKey` will match. The finding is reported as *probable* for exactly this reason ‚Äî
confirm the type is `Microsoft.Win32.Registry` before acting.

## Target platform

This is a **Windows lock-in**, not a gone-everywhere API. When you scan with a Windows target
(`--target net10.0-windows`), MigrationScan treats this finding as *satisfied*: it stays listed
for completeness but is excluded from the counts, effort estimate, and `--fail-on`. It only
becomes migration cost if the app must also run off Windows.
