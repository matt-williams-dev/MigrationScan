# MIG4008 — Thread.Abort

| | |
| --- | --- |
| **Category** | Runtime failures |
| **Severity** | Medium |
| **Effort** | Small |
| **Confidence tier** | 2 — Probable |

## What it detects

An argument-less `.Abort()` invocation (typically `Thread.Abort`).

## Why it matters

`Thread.Abort` is not supported on modern .NET and throws `PlatformNotSupportedException`.

## How to fix it

Use cooperative cancellation: pass a `CancellationToken` into the work and have it check the
token and return, instead of aborting the thread from outside.

## Known false positives

Tier 2. The rule matches any argument-less `.Abort()` call, so an unrelated method named
`Abort` (e.g. a transaction or animation API) will also match. Confirm the receiver is a
`System.Threading.Thread`.
