# MIG1010 — Vendored DLL with no source and no NuGet equivalent

| | |
| --- | --- |
| **Category** | Project and build |
| **Severity** | High |
| **Effort** | Large |
| **Confidence tier** | 1 — Certain |

## What it detects

A `<Reference>` with a `<HintPath>` that points at a **checked-in assembly** rather than a
restored NuGet package — a third-party DLL committed to the repository with no source and no
package feed behind it.

```xml
<!-- Flagged: vendored, checked-in path -->
<Reference Include="AxRANGERLib">
  <HintPath>libs\AxRANGERLib.dll</HintPath>
</Reference>

<!-- Not flagged: restored from the NuGet cache -->
<Reference Include="Newtonsoft.Json">
  <HintPath>..\packages\Newtonsoft.Json.13.0.3\lib\net45\Newtonsoft.Json.dll</HintPath>
</Reference>
```

A HintPath containing `packages/` or `.nuget` is treated as a normal package restore and is
**not** flagged (that surface is handled by MIG1002 / MIG2001).

## Why it matters

A vendored assembly is a migration unknown. Many are Framework-only, or ActiveX/COM interop
wrappers (like the `Ax*` control host generated by Visual Studio), and will not load on modern
.NET at all. Because there is no NuGet package to bump, each one needs a per-assembly decision:
does it load, is there a supported successor, or does it block the migration.

This is often a **Windows lock-in** rather than an outright dead end — an ActiveX/interop
wrapper may still work when the project targets `net-windows`. But if the vendor is defunct and
there is no equivalent, this can block the migration regardless of target.

## How to fix it

For each vendored assembly:

1. Try loading it on modern .NET (targeting `net-windows` first if it is Windows interop).
2. Look for a supported successor or an official NuGet package.
3. If neither exists, plan a replacement — this is the large-effort case that drives scope.

## Known false positives

The reference and its checked-in path are certain. Whether the assembly is actually a problem
is the part that needs assessment — a checked-in DLL that happens to already run on modern .NET
needs no work beyond confirming it loads.
